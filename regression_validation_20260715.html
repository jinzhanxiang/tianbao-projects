六、跨项目关键异常诊断

6.1 raw.githubusercontent.com CDN 压缩 bug（已记录）

现象： - gh api 列出的 innoshine_dd.html 大小：193,840 bytes - curl ?nocache=$(date +%s%N) 拉取大小：40,631 bytes（仅 21%）

根因： - CDN 节点边缘 gzip 压缩 - 加 ?nocache= 没解决，因为 CDN 节点本身对响应做压缩

解决方案： - 用 gh api download_url（实为 raw.githubusercontent）+ Accept-Encoding: identity 头 - 或用 gh api 直接下载文件

影响范围： - 仅影响离线回归测试拉取已部署文件 - 不影响真实用户访问（CDN 压缩是好事，节省带宽）

6.2 xingheng 同样的 CDN 问题

  • gh api 列出的 report_main.html 大小：345,630 bytes
  • 第一次 curl 拉取：39,229 bytes
  • 加 nocache 后拉取：345,630 bytes ✅
